**Module-level operator functions** (each takes population array, fitness array, rng, and operator-specific params):

Selection operators (select 2 parents from population):
- `tournament_selection(population, fitness, rng, *, k=3)`: Pick k random indices, return the one with lowest fitness. Repeat for 2 parents.
- `roulette_wheel_selection(population, fitness, rng)`: Transform fitness to positive selection weights via `weights = 1.0 / (fitness - fitness.min() + 1.0)` (handles minimization -- lower fitness = higher weight). Normalize to probabilities. Use rng.choice with p=probs to select 2 parents.
- `ranked_selection(population, fitness, rng)`: Rank individuals (best=highest rank N, worst=rank 1). Selection probability proportional to rank. Select 2 parents.

Crossover operators (take 2 parents as np.ndarray, produce 2 children):
- `npoint_crossover(parents, rng, *, n=1)`: Generate n sorted cut points. Alternate segments between parents. Return 2 children as np.ndarray.
- `uniform_crossover(parents, rng)`: For each gene, swap with probability 0.5. Return 2 children.

Mutation operator:
- `random_mutation(dna, bounds, rng, *, k=1)`: Randomly select k positions. For each, replace with random value within that position's bounds. Return mutated DNA as np.ndarray.

**Operator registries** (dicts mapping string names to functions):
```python
_SELECTION_REGISTRY = {'tournament': tournament_selection, 'roulette_wheel': roulette_wheel_selection, 'ranked': ranked_selection}
_CROSSOVER_REGISTRY = {'npoint': npoint_crossover, 'uniform': uniform_crossover}
```

**GA class:**
```python
class GA(DiscreteOptimizer):
    def __init__(self, fitness_fn, bounds, *,
                 population_size: int = 50,
                 selection: str | dict | Callable = 'tournament',
                 crossover: str | dict | Callable = 'npoint',
                 mutation_rate: float = 0.1,
                 mutation_k: int = 1,
                 elitism: int = 1,
                 seed=None, cache_size=None, record_history=False, progress=False):
```

Constructor (per D-05, D-06, CORE-06):
- Validate population_size >= 4 (need at least 2 parents + 2 children)
- Validate mutation_rate in [0, 1]
- Validate elitism >= 0 and < population_size
- Resolve selection operator via _resolve_operator (string->registry lookup, dict->registry lookup + extract params, callable->use directly)
- Resolve crossover operator similarly
- Store resolved callables and their params

`_resolve_operator(self, spec, registry: dict, name: str) -> tuple[Callable, dict]`:
- If callable(spec): return (spec, {})
- If isinstance(spec, str): lookup in registry, raise ValueError if not found listing valid options
- If isinstance(spec, dict): extract 'method' key, lookup in registry, remaining keys are params
- Else: raise TypeError

Override `run()` to initialize population before calling super():
- self._population: np.ndarray of shape (population_size, n_dims) -- random initialization within bounds
- self._fitness_values: np.ndarray of shape (population_size,) -- evaluate all initial individuals

`_step(self, iteration: int) -> dict | None`:
- Create next generation:
  - Keep top `elitism` individuals (elitist survival)
  - Fill remaining slots by repeating: select 2 parents -> crossover -> mutate each child (with mutation_rate probability) -> add to next gen
  - Clip all children to bounds
- Evaluate new individuals via self._evaluate (converts each row to tuple first)
- Update self._population and self._fitness_values
- If record_history: return {'best': float(self._fitness_values.min()), 'mean': float(self._fitness_values.mean()), 'std': float(self._fitness_values.std())}
